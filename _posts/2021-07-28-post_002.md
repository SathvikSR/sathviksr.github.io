## Setting up Vim and its basics.

![Vim Logo](https://upload.wikimedia.org/wikipedia/commons/thumb/9/9f/Vimlogo.svg/479px-Vimlogo.svg.png)

Whenever I search 'Getting started with programming', I mostly find tutorials and articles diving straight into the syntax of the language and few with the absolute basic setup to get coding. The first and the foremost tool you need is a text editor which offers you all the essential functionality you need.
Now everybody likes to brag about feature X of editor Y or how their editor is fast and offers customisations. But at the end of the day there is not a single software which is great at everything. But some do come close to it. These are editors like Emacs, SublimeText, Nano, Vi, Vim etc. (Sorry VSCode:(). Most of these are terminal based editors. The great thing about them is that they do not need heavy graphics to run, making them fast and efficient in comparison to their GUI counterparts.

In this post I write about setting up Vim on MacOS and run through a basic tutorial.

Vim is an improved version of the Vi text editor that dates back to the early days of Unix. Its is a full-fledged text editor which can meet all your needs and is heavily customisable. Although the barrier to mastery is high, it's quite rewarding to learn the editor and to impress your friends(or employers!).

### 1)Switching to bash.

The default shell of macOS as of BigSur is Zsh. But to get follow this tutorial we need to switch to Bash.
    $ chsh -s /bin/bash

Here "chsh" command refers to "change shell".
